How to read this document: This DFD describes all personal data flows between data subjects (users in Rwanda), the Bara Afrika platform, and third-party service providers outside Rwanda. All external communications use HTTPS / TLS encryption.

1. System Entities (Nodes)

Entity Type Location Description
User (Rwanda / Global) External entity (data subject) Rwanda (primarily) End-user accessing Bara Afrika via web browser. Includes registered users, businesses/sellers, event organizers/attendees, artists/creators.
Bara Afrika Web App (Browser / Client) Internal — frontend User's device (Rwanda) React/Next.js single-page application running in the user's browser. Communicates with Clerk for auth and Supabase for data.
Frontend Hosting (Vercel / CDN) External — hosting [Vercel global CDN / confirm hosting] Hosts the compiled web application. Serves static assets and the Next.js app. No personal data is stored long-term by the CDN.
Clerk (Auth Provider) External — vendor Germany (primary), Ireland (backup) — EU Manages user authentication, sessions, and identity. Holds: name, email, phone, Clerk user ID, session tokens.
Supabase (DB + Storage) External — vendor Zurich, Switzerland (eu-central-2) Managed PostgreSQL database and object storage. Holds: all platform data (profiles, UGC, events, listings, streams, logs).
Supabase Edge Function: send-email Internal — backend Runs within Supabase (Switzerland) Serverless function that receives email triggers from the database and calls the Resend API to send transactional emails.
Resend Email API External — vendor United States (AWS) Email delivery provider. Receives email address, subject, and transactional HTML content. Delivers emails to the user's inbox.
Admin User (Internal) Internal — privileged user Rwanda (Bara Afrika team) Admin staff accessing the platform admin interface. Has elevated access controlled by the admin_users table.

4. Boundaries and Scope

  • Rwanda boundary: User devices (browsers), Admin devices, the Bara Afrika team
  • Outside Rwanda: Clerk (Germany/Ireland), Supabase (Switzerland), Resend (USA), Frontend Hosting (CDN)
  • No bulk data exports: All data flows are API-based and tied to individual user sessions / platform events. No bulk data files are transferred outside Rwanda.
  • No analytics providers: No Google Analytics, GTM, Mixpanel, PostHog, or similar third-party analytics are currently integrated.
