2. Description of the Platform and Data Subjects

Bara Afrika is a digital platform that provides the following services to users in Rwanda and the region:

  • User accounts and profiles
  • Business and marketplace listings (reviews, discovery)
  • Events creation, promotion and ticketing
  • Music/content streams (play history, likes, playlists)
  • Blog and community content

The platform processes personal data relating to the following categories of data subjects:

  • Users / account holders (~1,000 projected Year 1)
  • Businesses / merchants and marketplace participants (~200 projected Year 1)
  • Event organizers and attendees (~100 projected Year 1)
  • Artists / content creators
  • Administrators (internal staff)

Personal data elements may include:

  • Identifiers and contact details: name, email address, phone number (where provided by user)
  • Account and authentication metadata: platform user identifiers (Clerk user ID), session metadata
  • User-generated content: reviews, listings, event submissions, profile images, blog posts
  • Operational logs and usage data: play history, likes, playlists, login timestamps, device/browser metadata as applicable

We do not intentionally collect special categories of sensitive personal data (health, biometric, political, religious, criminal records, etc.).

4. Reasons for Storing Personal Data Outside Rwanda

We request authorization to store personal data outside Rwanda for the following reasons:

  • The selected vendors provide reputable managed cloud infrastructure with established security controls, compliance certifications, and contractual data protection commitments
  • Supabase's Zurich region provides encryption at rest, encrypted backups, access controls, audit trails, and incident notification as documented in their DPA
  • Clerk's EU-based infrastructure (Germany/Ireland) offers robust identity and authentication services under GDPR governance, reducing risk and providing established security controls
  • Resend enables reliable transactional email delivery with contractual data processing safeguards
  • Local Rwanda-based alternatives offering equivalent managed infrastructure, security certifications, and service levels are not currently available
  • The use of these vendors enables availability, scalability, managed backups, and disaster recovery capabilities necessary for continuity of service

5. Safeguards and Security Measures

5.1 Measures implemented by Bara Afrika

  • Role-based access controls for all administrative access
  • Principle of least privilege — admin roles tiered (super_admin / admin / moderator)
  • Admin management actions logged in admin_activity_log audit table
  • Encrypted transport: all external traffic uses HTTPS / TLS
  • Supabase Row Level Security (RLS) applied to user data tables where applicable
  • Secrets management: API keys and credentials stored as environment variables, never in client-side code or source repository
  • Incident response process and breach reporting readiness (see Breach Response Playbook)
  • Data minimization: only data necessary for the service is collected; sensitive categories not requested

5.2 Measures implemented by service providers

  • Supabase: AES-256 encryption at rest, encrypted backups, least privilege access controls, audit trails, monitoring, security incident notification within 48 hours (per DPA) — see Supabase DPA attached
  • Clerk: EU-compliant identity infrastructure, GDPR-aligned data processing, access controls and security measures per their compliance documentation — see Clerk terms/DPA attached
  • Resend: Contractual data processing safeguards, secure email delivery infrastructure — see Resend terms/DPA attached
